1. Goal identification

  • client defines his requirements and vision
  • brief introduction of the phases in the software development process
  • goals are clearly set to ensure that the software solution precisely complies with the specific bussiness need

2. Strategic planning

  • identification and analysis of business and technical requirements
  • analysis of the business environment, competitors and identification of the end user profiles
  • development of priorities in case one of the goals is beyond budget

Scrum

  • agile methodology that iteratively delivers demanded features
  • each iteration results in working software that is ready for deployment
  • quick development of the first working version and inclusion of new emerging desires of the client

3. Design

  • definition of software architecture - design of software components and their interactions
  • flexible architecture enables our clients to easily modify software according to their current needs
  • designing the user interface with an emphasis on pleasant user experience and responsive design (our applications work on all devices)

4. Implementation and testing

  • creation of the user interfaces
  • back-end development that consists of the core application logic, databases, data and application integration
  • internal testing to ensure that software fulfills desired specifications

5. Deployment and delivery

  • deploying software to a test and production environment (installation on servers and/or devices)
  • acceptance testing by the customer
  • collection and processing of remarks, inclusion of the remarks in the subsequent iteration

7. Maintenance and support

  • initial training and familiarization tour for software
  • preventive, diagnostic, updating, and repair procedures
  • ongoing critical support
